# 论文输出审计 (PaperSpine Audit)

> 基于 paper-spine-audit 方法论，审计论文输出的完整性和质量

## 触发条件
用户说「检查论文」「审计」「check paper」「完整性检查」时加载。

## 审计清单

### 结构完整性
- [ ] Title Page 包含所有作者及单位
- [ ] Abstract 在字数限制内
- [ ] 所有图表在正文中引用
- [ ] 所有引用在参考文献列表中
- [ ] 无孤立引用（列表中有但正文未引用）

### 数据一致性
- [ ] Abstract 中的数字 = Results 中的数字
- [ ] Results 中的发现 = Discussion 中讨论的发现
- [ ] Table 中的数据 = 正文中引用的数据
- [ ] p 值报告格式一致

### 统计完整性
- [ ] 每个统计检验报告了：test statistic + p 值 + 效应量 + df
- [ ] 多重比较校正方法已描述
- [ ] 样本量在 Methods 和 Results 中一致

### 参考文献
- [ ] 格式一致（APA/Vancouver/Nature）
- [ ] 所有 DOI 可访问
- [ ] 自引比例是否合理（<20%）

### BCI/MEG/EEG 特定检查
- [ ] 设备型号和软件版本已报告
- [ ] 采样率和滤波器参数已报告
- [ ] 参考电极和接地方式已报告
- [ ] ICA 剔除成分数量已报告
- [ ] Epoch 拒绝标准和保留率已报告

## 审计报告模板
```markdown
## 论文审计报告

### 严重问题 🔴（必须修复）
- Abstract 中报告 N=32，Methods 中 N=30 → 不一致

### 遗漏 ⚠️（建议补充）
- Figure 3 未在正文中引用
- 未报告效应量（仅报告了 p 值）

### 格式问题 ✏️
- 参考文献 #15 格式不一致

### 通过项 ✅
- 12/15 检查项通过
```
